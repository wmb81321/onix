# Convexo P2P — Testing Practices

## Flow Tests

- **`flowA.spec.ts` and `flowB.spec.ts` use Stripe TEST MODE keys only.** Verify with `expect(process.env.STRIPE_SECRET_KEY).toMatch(/^sk_test_/)` at the top of every flow test. Never run against sandbox-but-production keys, never against live keys.
- **Stripe webhook signatures are real, not mocked.** Use `stripe.webhooks.constructEvent()` with payloads generated by the Stripe CLI (`stripe trigger`) or by signing fixtures with `stripe.webhooks.generateTestHeaderString`. Mocking the verifier defeats the security model.

## Solidity Tests

- **All Solidity tests run against a Tempo testnet fork before any mainnet deploy.** `forge test --fork-url $TEMPO_TESTNET_RPC_URL` is part of the deploy pipeline. A failing fork test blocks deploy.
- **Use `vm.expectEmit` before every state-changing call** in escrow tests — verify the expected event fires before the state mutation it records.
- **Cover the negative paths.** Reentrancy attempt, release without ATTESTED status, double-lock, expired refund — every revert path has an explicit test.

## Playwright

- **Stripe Link widget end-to-end runs headless.** `stripe.initLinkConnection()` is exercised through Playwright (`plugin:playwright` MCP) using a test Link account.
- The `stripeContext` shared between tests must be reset between cases — no cross-test state leakage.

## State Machine Idempotency

- **Every transition is tested for idempotency.** The test runs the same step twice and asserts: (a) state is unchanged after the second call, (b) no duplicate side-effect (no second payout, no second escrow release, no second x402 charge), (c) returned value is identical.
- Idempotency tests live in `agent/src/state/__tests__/idempotency.spec.ts` and run on every PR.

## Coverage Gate

- **Minimum 80% line coverage on `agent/src/`** before any deploy command (`/deploy-escrow`, `/register-agent`) proceeds. The `pre-deploy.sh` hook enforces this.
- Coverage exclusions are limited to `agent/src/lib/types.ts` and generated files. Excluding business logic to hit the gate is forbidden.

## What NOT to Mock

- Never mock the Stripe webhook signature verifier.
- Never mock the ERC-8004 `getValidationStatus` read.
- Never mock the escrow contract — use a fork or a deployed testnet instance.
